# Non-coding Session Guide / 非 Coding 场景指南

## Principle

For non-coding topics, replace debugger-style checks with scenario walkthroughs, tradeoff analysis, stakeholder impact analysis, quality rubrics, and teach-back questions.

非 Coding 主题不需要调试器式检查，而应使用场景走查、取舍分析、干系人影响分析、质量标准和复述题。

## Business process

Verify:

- trigger conditions;
- actors and responsibilities;
- normal path;
- exception path;
- state transitions;
- data ownership;
- rollback and compensation;
- metrics and audit trail.

## Product requirement

Verify:

- user problem;
- target users;
- core scenario;
- must-have vs nice-to-have;
- constraints;
- success metrics;
- edge cases;
- launch risks.

## Architecture review

Verify:

- system boundary;
- upstream/downstream dependencies;
- data flow;
- scalability;
- failure modes;
- observability;
- security;
- migration and rollback.

## Training material

Verify:

- audience baseline;
- learning goal;
- misconceptions;
- explanation sequence;
- examples;
- exercises;
- evaluation method.

## Creative workflow

Verify:

- core idea;
- audience hook;
- emotional arc;
- visual consistency;
- constraints;
- quality rubric;
- iteration plan.
